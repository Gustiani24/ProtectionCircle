About & security

ProtectionCircle is a read-and-write dashboard for the IronFist contract. All on-chain actions (open tunnel, bind session, close tunnel) require a connected wallet. The gatekeeper address can register exit nodes and pause the network; only the treasury and relay addresses are shown for reference.

This interface is not audited. Verify contract addresses and behavior before use on mainnet. IronFist uses immutable constructor addresses for gatekeeper, treasury, and relay.

Addresses shown in Network info are fixed at IronFist deployment and cannot be changed.

For production, always confirm you are interacting with the intended chain and contract.

Runbook & reference

IronFist steps

  1. Gatekeeper registers exit node (node_id, region, endpoint_hex)
  2. User opens tunnel (tunnel_id, owner, exit_node_id)
  3. User binds session (tunnel_id, session_id, client)
  4. Owner or gatekeeper closes tunnel (tunnel_id)
  5. Gatekeeper may pause or resume network

Example: open tunnel

open_tunnel("my-tunnel-1", "0xYourAddress...", "node-us-1")

Example: bind session

bind_session("my-tunnel-1", "sess-1", "0xClientAddress...")

Example: close tunnel (owner or gatekeeper)

close_tunnel("my-tunnel-1") # caller must be owner or gatekeeper

Example: register exit node (gatekeeper only)

register_exit_node("node-ap-1", "AP", "0x0c03") # gatekeeper only

Glossary

Gatekeeper — Address that can register exit nodes and pause/resume the network. Set at deployment.

Exit node — A registered routing endpoint with a region and endpoint identifier.

Tunnel — A channel opened by an owner and bound to one exit node. Can have multiple sessions.

Session — A client address bound to a tunnel. Used for routing or access control.

Error codes (IronFist)

Contract revert codes you may see when a call fails:

  • IF_ZERO_ADDR — Invalid or zero address
  • IF_NOT_GATEKEEPER — Caller is not gatekeeper
  • IF_NETWORK_PAUSED — Network is paused
  • IF_INVALID_NODE_ID — Exit node id empty or too long
  • IF_INVALID_REGION — Region empty or too long
  • IF_NODE_EXISTS — Exit node already exists
  • IF_NODE_CAP — Exit node limit reached
  • IF_NODE_MISSING — Exit node not found
  • IF_INVALID_TUNNEL_ID — Tunnel id empty or too long
  • IF_TUNNEL_EXISTS — Tunnel already exists
  • IF_TUNNEL_CAP — Tunnel limit reached
  • IF_TUNNEL_MISSING — Tunnel not found
  • IF_TUNNEL_CLOSED — Tunnel already closed
  • IF_INVALID_SESSION_ID — Session id empty or too long
  • IF_SESSION_CAP — Session limit per tunnel reached
  • IF_SESSION_EXISTS — Session already bound

Events (IronFist)

Emitted by the contract:

  • ExitNodeRegistered — node_id, region, endpoint_hex, at_block
  • TunnelOpened — tunnel_id, owner, exit_node_id, at_block
  • SessionBound — tunnel_id, session_id, client, at_block
  • TunnelClosed — tunnel_id, at_block
  • NetworkPaused — by, at_block
  • NetworkResumed — by, at_block
